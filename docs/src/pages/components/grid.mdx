# Grid

<ComponentInfo title="Grid" mode="description" />

Grid can be broken out into 2 groups of components, those for **Layout**, and those for
**Breakpoints**.

###### Grid Components
```jsx hideTitle
import {
    // Layout
    CenteredCell,
    Col,
    Container,
    Hidden,
    Row,
    ScreenBadge,
    ScreenClass,
    Visible,

    // Breakpoints
    CustomAdaptor,
    LegacyDesktop,
    LegacyTablet,
    LegacyPhone,
    XS,
    SM,
    MD,
    LG,
    XL,
    XXL,
    LegacyBreakpoints,
    StandardBreakpoints,
} from '@retailmenot/anchor';
```

---

## Layout Components

These are components that are built using
<a href="https://awesome-styled-grid.netlify.com/" target="_blank">react-awesome-styled-grid</a>,
which in itself is based off of
<a href="//material.io/design/layout/responsive-layout-grid.html#columns-gutters-margins" target="_blank">Google's Material Design</a>.
They provide structure to content and layout with utilities for responsive design. The components
that fit into this category are noted in the list below.

* `CenteredCell`: This layout component is not part of `react-awesome-styled-grid`. It can still be
used within the `Container` component as noted in the example below. It provides a column that
centers its children.
* `Col`: A column. Has the most options and props.
* `Container`: Containing element. All other layout components should be a child of this one.
* `Hidden`: Children of this component will be hidden by default unless at a specific breakpoint.
* `Row`: A row.
* `ScreenBadge`: This is strictly for debugging. It shows the breakpoint in the bottom-right corner
of the page.
* `ScreenClass`: A component whose render method exposes the breakpoint.
* `Visible`: Children of this component will be visible by default unless at a specific breakpoint.

The `debug` prop is assigned to nearly every component in the example. This shows the purple
background for each component and makes it easier to plan a layout.

###### Layout Components (Live Code Editor)
```tsx live hideTitle
<Container debug>
    <CenteredCell>
        <ScreenClass render={(screen) => <>{screen}</>} />
    </CenteredCell>
    <Row debug>
        <Col
            debug
            md={4}
            lg={6}
            xl={8}
            offset={{
                md: 2,
                lg: 3,
                xl: 2
            }}
        >
            Hello!
        </Col>
    </Row>
    <Row debug>
        <Hidden xs xl>
            <Col debug>
                1
            </Col>
        </Hidden>
        <Col debug>2</Col>
        <Visible md xs>
            <Col debug>
                3
            </Col>
        </Visible>
    </Row>
    <Row debug>
        <Col debug>
            4
        </Col>
        <Col debug>
            5
        </Col>
        <Col debug>
            6
        </Col>
    </Row>
    <ScreenBadge />
</Container>
```

## Breakpoint Components

These components are designed to help make changes to a layout on an even more granular level than
the Layout components can via breakpoints. The components that fit into this category are noted in
the list below.

* `CustomAdaptor`: Lets the user specify their own breakpoints at which point content should be
shown.
* `LegacyDesktop`: Uses the `LegacyBreakpoints` object to define at which point content should be
shown for desktop.
* `LegacyTablet`: Uses the `LegacyBreakpoints` object to define at which point content should be
shown for tablet.
* `LegacyPhone`: Uses the `LegacyBreakpoints` object to define at which point content should be
shown for tablet.
* `XS`: Uses the `StandardBreakpoints` object to define at which point content should be
shown for *xs*.
* `SM`: Uses the `StandardBreakpoints` object to define at which point content should be
shown for *sm*.
* `MD`: Uses the `StandardBreakpoints` object to define at which point content should be
shown for *md*.
* `LG`: Uses the `StandardBreakpoints` object to define at which point content should be
shown for *lg*.
* `XL`: Uses the `StandardBreakpoints` object to define at which point content should be
shown for *xl*.
* `XXL`: Uses the `StandardBreakpoints` object to define at which point content should be
shown for *xxl*.
* `LegacyBreakpoints`: An object containing min-max breakpoint values based on RetailMeNot's
legacy website.
* `StandardBreakpoints`: An object containing min-max breakpoint values based on newer standards.

<br />

### StandardBreakpoints

The components, `XS`, `SM`, `MD`, `LG`, `XL` and `XXL` are all based on the `StandardBreakpoints`
object. Below are the min and max values for each of those breakpoints, and an example of them in
use.

###### StandardBreakpoints
```jsx hideTitle
{
    xs: {
        max: 575,
    },
    sm: {
        min: 576,
        max: 767,
    },
    md: {
        min: 768,
        max: 991,
    },
    lg: {
        min: 992,
        max: 1199,
    },
    xl: {
        min: 1200,
        max: 1599,
    },
    xxl: {
        min: 1600,
    },
}
```

```tsx live
<div>
    <XS>
        XS
    </XS>
    <SM>
        SM
    </SM>
    <MD>
        MD
    </MD>
    <LG>
        LG
    </LG>
    <XL>
        XL
    </XL>
    <XXL>
        XXL
    </XXL>
</div>
```

### LegacyBreakpoints

The components, `LegacyDesktop`, `LegacyTablet`, and `LegacyPhone` are all based on the
`LegacyBreakpoints` object. Below are the min and max values for each of those breakpoints, and an
example of them in use.

###### LegacyBreakpoints
```jsx hideTitle
{
    desktop: {
        min: 1000,
    },
    tablet: {
        min: 899,
        max: 999,
    },
    phone: {
        max: 898,
    },
}
```

```tsx live
<div>
    <LegacyPhone>
        LegacyPhone
    </LegacyPhone>
    <LegacyTablet>
        LegacyTablet
    </LegacyTablet>
    <LegacyDesktop>
        LegacyDesktop
    </LegacyDesktop>
</div>
```

### CustomAdaptor

The `CustomAdaptor` allows for custom maximum and minimum widths per the example below.

```tsx live
<div>
    <CustomAdaptor
        maxWidth={399}
        onChange={(matches) => console.log(`onChange ${matches}`)}
    >
        Up to 399
    </CustomAdaptor>

    <CustomAdaptor
        minWidth={400}
        maxWidth={1000}
    >
        Between 400 & 1000
    </CustomAdaptor>

    <CustomAdaptor
        minWidth={1001}
    >
        Above 1001
    </CustomAdaptor>
</div>
```
